📄 main.py

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

@app.get("/status")
def status():
    return {"status": "running", "platform": "Replit"}

class GenRequest(BaseModel):
    prompt: str

@app.post("/generate")
def generate(req: GenRequest):
    return {"response": f"Generated response for: {req.prompt}"}

# Replit FastAPI Backend

Live FastAPI backend designed for Replit and synced with GitHub.

## Routes

- `GET /status`: Health check
- `POST /generate`: Generate text from input

## Deployment

- Replit auto-installs `requirements.txt`
- Hosted on `https://<your-repl>.repl.co`
- GitHub syncing enabled via Replit Git integration

  push:
    branches: [ main ]

jobs:
  push:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3
      - name: Log commit
        run: echo "Pushed from Replit at $(date)"

✅ 2. HOW TO DEPLOY FROM REPLIT TO GITHUB
	1.	Create a new GitHub repo (e.g. replit-fastapi-backend)
	2.	Go to Replit:
	•	New Repl → “Import from GitHub” → paste repo link
or
	•	Create a new Repl → paste in this code → go to Git tab
	3.	Git Tab Setup in Replit:
	•	Connect your GitHub account
	•	Select your repo
	•	Replit will auto-sync changes
	•	Use Git tab to commit → push to GitHub

⚠️ If Git tab doesn’t show up, click the 3-dot menu → Enable Git
